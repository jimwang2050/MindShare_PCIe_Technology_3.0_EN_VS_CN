## **数据流和数据块 (Data Stream and Data Blocks)**

链路通过发送 SDS 有序集合并转换到 L0 链路状态来进入数据流。在数据流中，多个数据块被传输，直到数据流以 EDS 令牌结束（除非错误提前结束它）。EDS 令牌总是占据有序集合之前的数据块的最后一个 4 个字符的位置。对 Skip 有序集合有一个例外，因为只要满足稍后讨论的某些条件，它们就不会中断数据流。

## **数据块帧构造 (Data Block Frame Construction)**

数据块由 TLP、DLLP 和用于传递信息的令牌 (Tokens) 组成。在数据块中还使用五种类型的数据结构（称为令牌）。每个都有便于接收器检测的模式。其中三个令牌可以在块的开头发送（即，紧跟在 Sync 数据块之后）。这些包括：

- Start TLP (STP) - 后跟一个 TLP

- Start DLLP (SDP) - 后跟一个 DLLP

- Logical Idle (IDLA) - 在没有数据包活动时发送

其余的令牌在数据块的末尾传递：

- End of Data Stream (EDS) - 在转换到有序集合之前

- End Bad (EDB) - 报告已检测到作废的数据包

## **成帧令牌 (Framing Tokens)**

规范定义了五个成帧令牌（或简称"令牌"），它们允许出现在数据块中，为了方便起见在第 417 页的图 12-8 中重复。五个令牌是：

**1. STP - Start TLP**：与早期版本非常相似，但现在包括整个数据包的 dword 计数。

**2. SDP - Start DLLP**

**3. EDB - End Bad**：用于以与早期 Gen1 和 Gen2 设计相同的方式作废 TLP，但现在连续发送四个 EDB 符号。END（End Good）符号现在被取消；如果没有明确标记为坏，则假定 TLP 是好的。

**4. EDS - End of Data Stream**：数据流的最后一个 dword，指示将至少跟随一个有序集合。奇怪的是，数据流实际上可能不会被此事件结束。如果跟随它的有序集合是 SOS 并且紧跟着是另一个数据块，则数据流继续。如果跟随 EDS 的有序集合是除 SOS 之外的任何东西，或者 SOS 之后没有数据块，则数据流结束。

**5. IDL - Logical Idle**：空闲令牌只是在链路逻辑空闲状态期间发送的数据零字节，此时没有 TLP 或 DLLP 准备好传输。

规范中显示令牌的方式与第 417 页的图 12-8 中显示的方式之间的区别在于，此图以小端序显示字节和位，而不是规范中使用的大端序位表示。之所以以这种方式显示，是为了说明位实际上出现在通道上的顺序。

## **发送器成帧要求 (Transmitter Framing Requirements)**

要开始此讨论，首先定义几件事会很有帮助。首先，回想一下数据流从 SDS 之后的第一个字符开始，并且可以包含由令牌、TLP 和 DLLP 组成的数据块。数据流在除 SOS 之外的有序集合之前的最后一个字符处完成，或者当检测到成帧错误时完成。在数据流期间，除了 SOS 之外不能发送任何有序集合。

其次，由于成帧问题通常会导致成帧错误，解释在这种情况下会发生什么会有所帮助。当发生成帧错误时，它们

**417**

## **PCI Ex ress Technolo p gy**

被视为接收器错误并按此报告。接收器停止处理正在进行的数据流，并且只有在看到 SDS 有序集合时才会处理新的数据流。作为对错误的响应，通过将 LTSSM 从 L0 引导到 Recovery 状态来启动恢复过程。期望这将在物理层中解决，不需要上层采取任何操作。此外，规范指出，预期从两个端口进入 Recovery 开始完成此往返所花费的时间应小于 1 μs。

现在，有了这些背景，让我们继续讨论成帧要求。在数据流中，发送器必须遵守以下规则：

- 发送 TLP 时：

   - STP 令牌必须紧跟着从链路层传递的 TLP 的全部内容，即使它被作废。

   - 如果 TLP 被作废，则 EDB 令牌必须出现在 TLP 的最后一个 dword 之后，但不得包括在 TLP 长度值中。

   - STP 在链路上的每个字符时间不能发送超过一次。

- 发送 DLLP 时：

   - SDP 令牌必须紧跟着从数据链路层传递的 DLLP 的全部内容。

   - SDP 在链路上的每个字符时间不能发送超过一次。

- 在数据流中发送 SOS (SKP 有序集合) 时：

   - 在当前数据块的最后一个 dword 中发送 EDS 令牌。

   - 将 SOS 作为下一个有序集合块发送。

   - 在 SOS 之后立即发送另一个数据块。数据流从随后这个数据块的第一个字符恢复。

   - 如果安排了多个 SOS，它们不能像在早期代中那样背靠背。相反，每个 SOS 必须以一个以 EDS 令牌结束的数据块作为前缀。数据块可以用 TLP、DLLP 或 IDL 填充。
